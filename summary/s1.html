# 总结 最近一个 项目 所用的新知识

总结最近在vue 项目开发中学到和用到的新知识，以便之后查看

# 1. 在 Vue 中新建标签页

大家都知道，vue 秉持的是SPA，也就是单页面复应用，一般不会又跳转一个新的页面的行为。 本次项目业务中就存在，点击之后进入另一个页面进行操作

  • Api：router.resolve(location, current?, append?) vue-router 官方文档
    这里我们只用第一个参数： location ：一个描述 目标位置的对象/路由对象

参数 location：

路由常用属性包括：
route.path  //路径
route.params //动态片段和全匹配片段，
route.query //表示 URL 查询参数
route.name //组件命名

返回值

const resolved: {
  location: Location;
  route: Route;
  href: string;
} = router.resolve(location, current?, append?)

而返回值的href 可以直接使用 window.open 打开，即可新建一个页面

const { href } = this.$router.resolve({
    path: `/task/${this.tableType}/${name}`, //进入任务详情页
});
window.open(href, '_blank'); //_blank 打开一个新的页面

需要注意的是 ：在新的页面与原页面，共享 全局的 router，但是 Vuex 的数据无法共享（当然可以跨页的性质，cookie 、localstorage 可以共享）

# 2. 分离 el-dialog

在 ElementUi 中，el-dialog 是一个对话框，一般会将它和父容器分离。例如页面上有很多展示信息，其中有一个信息可以修改，至于具体如何修改，我们用 el-dialog 去做，这时将它和外层进行分离是一个明智的选择：

//父组件
<AllocateDialog
    :dialogVisible.sync="allocateVisible"
    ref="all_dia"
    @allocate="allocate"
    taskType="d"
></AllocateDialog>

<script>
    export default {
        data() {
            return {
                allocateVisible: false,
            };
        },
    };
</script>

//子组件
<template>
    <el-dialog
        :visible="dialogVisible"
        @update:visible="$emit('update:dialogVisible', false)"
    ></el-dialog>
</template>

解释 一旦分离 el-dialog 组件,由于组件嵌套层次过多，.sync 修饰符 就不能再继续使用 ，只能手动的将更深层次组件 对 visible 的变化 反应到 el-dialog 组件上， 并继续向上传递 当点击 对话框遮罩层，或者点击 关闭图标后，el-dialog 的内部组件会向上 emit 一个参数为update：visible,false 的事件，而 el-dialog 组件，必须继续向上 emit，才能正确关闭 对话框

# 4. localstorage 和 vuex 结合

vuex 一旦刷新就会失去保存的状态，是很让人头疼的事情，但可以借助 localstorage 或者 sessionstorage/cookie 等 浏览器存储 与 vue 配合来存储状态

主要关键点是：监听组件的 mounted 生命周期 因为，页面刷新后首先触发的 是 mounted 事件，在此时，把存储在 sessionstorage 的数据保存到 vuex 中

//home.vue 组件内
this.$store.dispatch('reqDangerTypes'); //刷新重新获取终端人员
this.$store.dispatch('reqStaffs');

//vuex actions 中
//第一次记载页面/页面刷新 后 主动获取 风险类型
 actions: {
        async reqDangerTypes({ commit }) {
            return new Promise(async (resolve, reject) => {
                const res = await getDangerTypes({
                    page: 1,
                    limit: 9999,
                });
                if (!res.flag) {
                    console.log('隐患类型获取失败');
                    return reject('failed');
                }
                commit('getDangerTypes', res.riskTypes);
                return resolve('success');
            });
        },
 }

# 5. 捕获 axios请求的错误

axios 或者axios.create 的实例返回的都是一个promise, 如果不捕获错误的话，控制台就会报红色的错 可以使用.catch ,将报错信息和状态码进行封装， 返回给调用者/await

//request.js
const errFun = (err) => {
    return {
        status: err.response.status,//状态码
        err,//错误信息
    };
};

export function deleteUser(enterpriseName, name) {
    return ins
        .post(
            '/user/delete',
            qs.stringify({
                enterpriseName,
                name,
            })
        )
        .catch(errFun);
}

# 6. post 请求时，是否要受用qs 进行序列化

项目起初，使用axios.post 请求，状态码一直是400，后台页接受不到 参数，后来发现时传参的问题，使用qs.stringify() 序列化，参数对象，后台就可以正常的接受到参数了 原因在于

  • axios 的 Content-Type 默认为 application/json
    这种类型的Content-Type 会把参数放在 Request 的 body 里

控制台Network截图

后台就要到RequestBody里面取参数（比如express 就是：req.body.xxx）

  • 而qs.stringify() 序列换后的 Content-Type 为 application/x-www-form-urlencodedbody ，也就是常说的 表单formdata

控制台Network截图 urlencoding (编码)后是 account=xxx&password=xxx 后台就要直接通url取参数（比如express 就是：req.params.xxx）

实际上是否需要用qs去序列化参数完全取决于后端要怎么接受数据

# 7. 对话框的 封装

在实际业务中，总会遇到使用对话框的情况，而每次都封装一遍很麻烦，特别是footer 的两个按钮每次都要写一遍插槽会显得很麻烦

<template>
<!--通过$attrs和 $listeners 获取父组件绑定但本组件没有声明的props 和事件，传递给el-dialog -->
    <el-dialog v-bind="$attrs" v-on="$listeners">
        <slot>对话框没有内容</slot>
        <!--向父组件暴露 footer 插槽,提供二次自定义的功能-->
        <slot name="footer" v-if="showFooter">
            <!--填充el-dialog 的footer 插槽-->
            <span slot="footer">
                <el-button @click="$emit('dialog-cancel')" size="medium"
                    >取 消</el-button
                >
                <el-button
                    type="primary"
                    @click="$emit('dialog-confirm')"
                    size="medium"
                    >确 定</el-button
                >
            </span>
        </slot>
    </el-dialog>
</template>

<script>
//MK 组件
export default {
    name: 'MKDialog',
    props: {
        showFooter: {
            type: Boolean,
            default: true,
        },
    },
    data() {
        return {};
    },
    inheritAttrs: false, //默认策略： 把未声明再props 里的属性绑定，转换为html 属性，阻止这个策略
};
</script>

<style></style>

关键：通过 $attrs和 $listeners 获取父组件绑定但本组件没有声明的props 和事件，传递给el-dialog进行绑定，这样可以支持多维度的属性绑定，不局限于通过props声明的属性

# 9.自定义滚动条

某些自定义的场景下，需要我们自定义页面的滚动条
对于webkit 内核的浏览器 可以使用 伪元素-webkit-scrollbar ，-webkit-scrollbar-thumb，-webkit-scrollbar-track 分别设置滚动栏，滑块，滚动轨道 的样式

html::-webkit-scrollbar {
    /*滚动条整体样式*/
    width: 8px;
    /*高宽分别对应横竖滚动条的尺寸*/
    height: 1px;
}

html::-webkit-scrollbar-thumb {
    /*滚动条里面小方块*/
    border-radius: 10px;
    box-shadow: inset 0 0 5px rgba(0, 0, 0, 0.2);
    background: #535353;
}

html::-webkit-scrollbar-track {
    /*滚动条里面轨道*/
    box-shadow: inset 0 0 5px rgba(0, 0, 0, 0.2);
    border-radius: 10px;
    background: #EDEDED;
}
